## 📌 Project Overview
This project focuses on building a robust end-to-end NLP pipeline for text classification. Using the **SMS Spam Collection Dataset**, the goal is to accurately classify messages as either **Ham** (legitimate) or **Spam**. 

The project demonstrates essential NLP techniques including text cleaning, tokenization, stemming, lemmatization, and vectorization, followed by a machine learning model for classification.

---

## 🎯 Learning Outcomes & Objectives
- **Apply NLP Preprocessing:** Implement tokenization, stopword removal, stemming, and lemmatization.
- **Text Vectorization:** Convert raw text into numerical data using **TF-IDF** and **CountVectorizer**.
- **Model Building:** Train a **Multinomial Naive Bayes** classifier for text categorization.
- **Evaluation:** Measure performance using Accuracy, Precision, Recall, and Confusion Matrices.

## 📂 Dataset Information
The dataset used is the **SMS Spam Collection Dataset**, which contains 5,574 SMS messages in English, tagged according to being ham (legitimate) or spam.

- **Source:** [Kaggle - SMS Spam Collection Dataset](https://www.kaggle.com/datasets/uciml/sms-spam-collection-dataset)

---

## 🚀 Workflow
1. **Data Cleaning:** Removing special characters, numbers, and converting text to lowercase.
2. **Preprocessing:** - **Tokenization:** Breaking text into individual words.
   - **Stopword Removal:** Filtering out common words (the, is, at) that don't add semantic value.
   - **Stemming/Lemmatization:** Reducing words to their root forms.
3. **Vectorization:** Transforming text into numerical features using TF-IDF.
4. **Classification:** Training a Multinomial Naive Bayes model.
5. **Evaluation:** Analyzing model results with a classification report and confusion matrix.
